🍱 우리가 만들 앱: 점심 & 음료 신청 앱

💡
이런 앱이에요 매일 점심시간 전에 팀원들이 메뉴와 음료를 신청하면, 한 화면에 신청 현황이 모여서 보이는 앱이에요.
  • 이름 입력 + 점심 메뉴 선택
    불고기덮밥 / 김치찌개 / 파스타 / 샐러드 중 하나 선택
  • 음료 선택 후 "신청하기" 클릭
    아메리카노 / 라떼 / 녹차 / 물 중 하나 선택
  • 오늘 신청 현황 실시간 확인
    누가 무엇을 신청했는지 목록으로 표시
  • 관리자 대시보드 — 메뉴별·음료별 집계
    불고기덮밥 5명 / 파스타 3명 / 아메리카노 7명 등 한눈에 확인

✅ 이 앱의 특징

인터넷이 되는 어디서든 접속 가능 · 팀 전체가 같은 URL로 사용 · 신청 내역은 Firebase에 자동 저장

📋 딱 3개 계정만 있으면 됩니다

💡
모두 무료! 설치할 프로그램 없어요. Google 계정 하나로 나머지도 쉽게 만들 수 있어요.
🐙
GitHub 가입 화면
github.com 접속 → 우측 상단 "Sign up" 클릭 → 이메일과 비밀번호 입력
↑ GitHub 가입 화면 — Sign up을 클릭해서 계정을 만드세요

✅ 3개 계정 모두 준비했다면 시작!

아래 "다음" 버튼을 눌러 앱 만들기를 시작하세요.

Step 1

🤖 AI Studio로 앱 코드 만들고 GitHub에 저장

AI에게 앱 설명을 붙여넣으면 코드가 자동으로 완성돼요.
완성되면 AI Studio에서 바로 GitHub에 저장합니다. ZIP 다운로드나 터미널 없이요!

⏱️ 약 20분

① Google AI Studio 열기

  1. 아래 버튼으로 Google AI Studio를 새 탭에서 여세요

    Google AI Studio 열기 →
  2. Google 계정으로 로그인 (이미 로그인되어 있으면 바로 진행)
    🤖
    Google AI Studio 메인 화면
    로그인하면 "Create new prompt", "Build", "My library" 등 메뉴가 보여요
    ↑ 이런 화면이 보이면 로그인 성공!
  3. 왼쪽 메뉴에서 Build 클릭 → Create app 클릭
    📱
    AI Studio — Build 탭
    왼쪽 메뉴에서 "Build"를 클릭하면 새 채팅 화면이 열려요.
    오른쪽에 앱 미리보기 화면이 함께 나타납니다.
    ↑ Build 탭 진입 — "Skip" 클릭 후 채팅창 사용 시작

② 모델 선택: Gemini Flash

  1. 화면 오른쪽 위 모델 선택 드롭다운 클릭
    ⚡
    모델 선택 드롭다운
    오른쪽 상단에 현재 모델명이 표시돼요 (예: "Gemini 2.0 Pro")
    클릭하면 모델 목록이 펼쳐집니다
    ↑ AI Studio Build 화면 — 상단 오른쪽에서 모델 선택
  2. 목록에서 Gemini 2.0 Flash 또는 Gemini Flash 선택
    Flash 모델이 빠르고, MVP에 딱 맞아요

③ 앱 설명 붙여넣기

📋
아래 내용을 그대로 복사해서 붙여넣으세요 수정 없이 그대로 넣어도 됩니다. 나중에 채팅으로 바꿀 수 있어요.
AI Studio Build 채팅창에 붙여넣기 
직장인용 점심 & 음료 신청 앱을 만들어줘.

점심 메뉴: 불고기덮밥, 김치찌개, 파스타, 샐러드
음료: 아메리카노, 라떼, 녹차, 물

화면 2개:
1. 신청 화면: 이름 + 메뉴/음료 선택 + "신청하기" 버튼 → Firebase 저장
2. 관리자 화면: 오늘 전체 신청 목록 + 메뉴별/음료별 집계 숫자

디자인: 깔끔한 흰색 배경, 한국어 UI, 모바일 최적화

Firebase 설정은 나중에 알려줄게. 지금은 아래처럼 빈 값으로 코드 작성해줘:
const firebaseConfig = {
  apiKey: "FILL_IN",
  authDomain: "FILL_IN",
  projectId: "FILL_IN",
  storageBucket: "FILL_IN",
  messagingSenderId: "FILL_IN",
  appId: "FILL_IN"
};
  1. 위 내용을 복사해서 AI Studio 채팅창에 붙여넣고 엔터(전송)
  2. AI가 코드를 작성합니다 (1~2분 대기)
    오른쪽 미리보기에 앱 화면이 나타나면 완성!
    ✨
    AI가 앱을 만드는 중...
    왼쪽 채팅에 코드가 생성되고, 오른쪽에 앱 미리보기가 나타나요.
    메뉴 선택 폼과 신청 목록이 보이면 성공!
    ↑ 오른쪽에 앱 미리보기가 나타나면 완성!

💬 마음에 안 드는 부분? GAS 채팅창에 바로 요청하세요!

  • "배경을 연한 파란색으로 바꿔줘"
  • "신청 버튼을 더 크게 만들어줘"
  • "관리자 화면에 날짜 필터도 추가해줘"
  • "집계 숫자 옆에 그래프도 표시해줘"

④ GitHub에 저장 (AI Studio 버튼 사용)

🚀
ZIP 다운로드 없이 바로 저장! AI Studio 오른쪽 위에 GitHub 연결 버튼이 있어요. 클릭 한 번으로 코드가 GitHub에 저장됩니다.
  1. AI Studio Build 화면 오른쪽 위에서 GitHub 아이콘 또는 Export to GitHub 버튼 클릭
    📂
    AI Studio 오른쪽 상단 — GitHub 버튼
    오른쪽 위에 GitHub 고양이 아이콘(🐙) 또는 "GitHub에 저장" 버튼이 있어요.
    찾기 어려우면 채팅창에 "GitHub에 업로드해줘" 라고 요청해도 됩니다!
    ↑ AI Studio 오른쪽 위 — GitHub 저장 버튼
  2. 처음 사용 시 → GitHub 계정 연결 클릭 → GitHub 로그인 후 권한 허용
    🔗
    GitHub 계정 연결 화면
    "Authorize Google AI Studio" 버튼이 나타나요.
    클릭하면 AI Studio와 GitHub이 연결됩니다. (처음 1회만)
    ↑ "Authorize" 클릭 → AI Studio와 GitHub 연결 (처음 1회만)
  3. 저장소(Repository) 이름 입력: lunch-app → Create and push 클릭
    📝
    새 저장소 이름 입력
    저장소 이름을 영어로 입력하고 "Create and push" 버튼을 클릭하세요.
    잠깐 기다리면 코드가 GitHub에 저장됩니다!
    ↑ "lunch-app" 입력 → Create and push 클릭

✅ 성공 확인

GitHub에서 lunch-app 저장소가 생기고, package.json, src/ 등이 보이면 완성!

✅ Step 1 완료 체크

Step 2

🔥 Firebase — 데이터 저장소 만들기

Firebase는 Google이 제공하는 무료 데이터베이스예요.
신청 내역이 여기 저장되고, 팀 전체가 실시간으로 볼 수 있어요.

⏱️ 약 15분

① Firebase 프로젝트 만들기

  1. 아래 버튼으로 Firebase 콘솔을 여세요

    Firebase 콘솔 열기 →
    🔥
    Firebase 홈 화면
    firebase.google.com 접속 후 "콘솔로 이동" 클릭
    ↑ Firebase 홈 — 우측 상단 "콘솔로 이동" 클릭 → Google 로그인
  2. 프로젝트 추가 클릭 (또는 "+ Add project")
    ➕
    프로젝트 추가 버튼
    로그인 후 화면 중앙에 큰 "프로젝트 추가" 카드가 있어요. 클릭하세요.
    ↑ Firebase 콘솔 → "프로젝트 추가" 클릭
  3. 프로젝트 이름 입력: lunch-app
    영어 소문자와 하이픈만 사용 가능해요
  4. Google Analytics → 지금은 사용 안 함 선택
    → 프로젝트 만들기 클릭 → 완료 후 계속

✅ 성공 확인

왼쪽에 메뉴가 있는 프로젝트 대시보드가 나타나면 완성!

② Firestore 데이터베이스 만들기

  1. 왼쪽 메뉴 → 빌드 → Firestore Database 클릭
    📊
    왼쪽 메뉴 — 빌드 → Firestore Database
    왼쪽 메뉴에서 "빌드"를 클릭하면 하위 메뉴가 펼쳐져요.
    그 안에 "Firestore Database"가 있습니다. 클릭하세요.
    ↑ 빌드 → Firestore Database → 데이터베이스 만들기
  2. 데이터베이스 만들기 버튼 클릭
  3. 테스트 모드로 시작 선택 → 다음
    ⚠️ 테스트 모드는 30일 후 만료돼요. 나중에 보안 설정을 바꿔야 합니다.
  4. 위치 선택: asia-northeast3 (서울) → 완료

✅ 성공 확인

"데이터" 탭이 있는 Firestore 화면이 나타나면 완성!

③ Google 로그인 켜기

  1. 왼쪽 메뉴 → 빌드 → Authentication 클릭 → 시작하기
  2. Sign-in method 탭 → 공급자 목록에서 Google 클릭
    🔐
    Authentication — Sign-in method 탭
    공급자 목록에 "Google"이 보여요. 클릭하면 설정 패널이 열립니다.
    ↑ Sign-in method → Google 클릭 → 사용 설정 ON → 저장
  3. 사용 설정 토글 → ON으로 변경 → 지원 이메일 선택 → 저장

✅ 성공 확인

Google 항목이 "사용 설정됨"으로 바뀌면 완성!

④ Firebase 설정값 복사 → AI Studio에 붙여넣기

💡
Vercel 환경변수 입력 필요 없어요! 설정값을 GAS 채팅창에 넣으면 AI가 코드에 직접 넣어줍니다. Vercel 설정이 훨씬 간단해져요.
  1. Firebase 콘솔 → 왼쪽 상단 톱니바퀴(⚙️) → 프로젝트 설정
  2. 아래 스크롤 → "내 앱" → 웹 아이콘 </> 클릭 → 앱 닉네임 입력(아무거나) → 앱 등록
  3. 화면에 나타난 firebaseConfig 코드 블록 전체 복사
    📋
    Firebase 설정 코드 화면
    프로젝트 설정 → 내 앱 아래에 firebaseConfig 코드가 나타나요.
    블록 전체를 드래그해서 복사하세요.
    ↑ 이 코드 블록 전체 복사!
    이렇게 생긴 코드가 나와요 — 전체 복사! 
    const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "lunch-app.firebaseapp.com",
  projectId: "lunch-app",
  storageBucket: "lunch-app.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123:web:abc..."
};
  4. AI Studio Build 탭으로 돌아가서 채팅창에 아래처럼 입력하세요:

    GAS 채팅창 입력 
    Firebase 설정값을 아래로 교체해줘. 그리고 GitHub에 다시 커밋해줘.

(여기에 복사한 firebaseConfig 붙여넣기)
    AI가 코드에 설정값을 채워 넣고 GitHub에 자동으로 커밋해줘요.

✅ 완료! 이제 Vercel에서 환경변수 입력이 필요 없어요

GitHub 저장소에 Firebase 설정이 포함된 최신 코드가 올라갔습니다.

✅ Step 2 완료 체크

Step 3

🚀 Vercel로 팀에 배포하기

GitHub에 올린 코드를 Vercel에 연결하면
인터넷 주소(URL)가 생겨서 팀 전체가 접속할 수 있어요.

⏱️ 약 10분

① GitHub 저장소를 Vercel에 연결

  1. 아래 버튼으로 Vercel을 새 탭에서 여세요

    Vercel 대시보드 열기 →
    ▲
    Vercel 대시보드
    로그인 후 대시보드가 보입니다
    ↑ Vercel — GitHub 계정으로 로그인 후 대시보드
  2. Add New... → Project 클릭
    ➕
    Add New Project
    대시보드 오른쪽 위에 있는 "Add New..." 버튼을 클릭하고, "Project"를 선택하세요.
    ↑ Vercel → "Import Git Repository" → Continue with GitHub
  3. GitHub 저장소 목록에서 lunch-app 찾기 → Import 클릭
    📂
    저장소 목록에서 lunch-app 선택
    Step 1에서 만든 "lunch-app" 저장소가 목록에 보여요.
    옆에 있는 "Import" 버튼을 클릭하세요.
    ↑ 저장소 목록에서 lunch-app → Import 클릭
  4. 빌드 설정 그대로 두고 → Deploy 버튼 클릭!
    ✅
    환경변수 입력 불필요! Step 2에서 GAS에 Firebase 설정값을 직접 넣었기 때문에 Vercel에서 별도로 입력할 것이 없어요.
  5. 1~2분 기다리면 "Congratulations! 🎉" 화면이 나타납니다
    화면에 표시된 URL이 우리 앱 주소예요 (예: lunch-app.vercel.app)
    🎉
    배포 완료 화면
    "Congratulations!" 메시지와 함께 앱 URL이 표시돼요.
    클릭하면 바로 앱이 열립니다!
    ↑ Congratulations! — 앱 URL을 클릭해서 확인하세요

✅ 성공 확인

URL을 클릭했을 때 점심 신청 앱 화면이 보이면 완성!

③ Firebase에 Vercel 주소 등록 (Google 로그인 오류 방지)

⚠️
이 단계를 빠뜨리면 배포된 앱에서 Google 로그인이 안 됩니다!
  1. Firebase 콘솔 → Authentication → Settings 탭
  2. 승인된 도메인 → 도메인 추가 클릭
    🌐
    Firebase — 승인된 도메인 설정
    Authentication → Settings 탭 → 승인된 도메인 섹션에 "도메인 추가" 버튼이 있어요.
    ↑ "도메인 추가" 클릭 → Vercel URL 입력
  3. Vercel에서 받은 URL 입력 (예: lunch-app.vercel.app) → 추가

✅ 이제 Google 로그인도 정상 작동해요!

배포된 URL에서 "Google로 로그인" 버튼이 정상 작동하면 완성!
